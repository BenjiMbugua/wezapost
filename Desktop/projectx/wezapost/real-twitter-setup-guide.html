üöÄ Step-by-Step OAuth Setup

1Get Twitter Developer Access

You need a Twitter Developer account (free but requires approval):

  • Go to Twitter Developer Portal
  • Apply for developer access (usually approved within hours)
  • Explain your use case: "Personal social media management tool"

2Create a Twitter App

Once approved, create a new app:

  1. Click "Create App" or "+ Add App"
  2. Fill in app details:
    • App Name: "WezaPost Social Manager" (or your preference)
    • Description: "Personal social media management and posting tool"
    • Website: http://localhost:3001 (for development)
  3. Set App Type to "Web App"
  4. Set permissions to "Read and Write"

3Configure OAuth Settings

Set up the callback URL in your Twitter app:

Callback URL:

4Get Your Credentials

From your Twitter app dashboard, copy these credentials:

Client ID: Usually starts with a long alphanumeric string
Client Secret: A secret key that should be kept private

üéØ Enter Credentials in WezaPost

Two Methods Available:

Method 1: Setup Wizard (Recommended)
  1. Go back to your WezaPost dashboard
  2. Click "Connect" next to Twitter
  3. The setup wizard should appear automatically
  4. Follow the step-by-step instructions
  5. Enter your Client ID and Secret in the secure form
Method 2: Environment Variables (Alternative)

If the wizard doesn't work, you can add to your .env.local file:

Replace with your actual credentials

üîß Troubleshooting

If Connection Fails:

  • Check that callback URL is exactly correct
  • Verify Client ID and Secret are properly copied
  • Make sure Twitter app has "Read and Write" permissions
  • Try the setup wizard again

Common Issues:

  • 400 Error: Usually incorrect callback URL
  • 401 Error: Invalid credentials
  • 403 Error: App permissions issue
